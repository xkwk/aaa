LLM-native actuarial workbench

How an actuary works with the AI agent

The actuary gives intent, evidence, and judgment. GitHub Copilot coordinates agents, skills, MCP tools, the wiki, Snowflake, and reusable Python capabilities.

Operating Pattern

Copilot is the control surface and reasoning layer. MCP tools are bounded capabilities; they do not decide actuarial intent.

A

Actuary

Provides the business question, source files, review comments, and professional judgment.

C

Copilot Orchestrator

Interprets intent, checks memory, chooses skills, delegates to specialists, and plans the next move.

S

Subagents + Skills

Bring specialist reasoning for profiling, wiki curation, Snowflake, dashboards, review, and tool building.

M

MCP Capabilities

Expose safe Python actions: convert, profile, search/write wiki, inspect Snowflake, run tests, record feedback.

O

Outputs

Updated wiki, proposed Snowflake objects, dashboards, tested Python tools, memos, and unresolved questions.

Human and Agent Responsibilities

Actuary owns judgment

The human remains accountable for actuarial meaning, materiality, and sign-off.

  • Frames the question and business context.
  • Supplies source material and explains constraints.
  • Corrects definitions, assumptions, and interpretation.
  • Approves methods, outputs, and communication.

Agent owns maintenance and execution

The AI handles the tedious work that makes knowledge and tools compound.

  • Profiles unfamiliar files and converts documents for review.
  • Maintains the markdown wiki, index, log, and open questions.
  • Builds or updates bounded Python tools when repeatability matters.
  • Runs tests, records feedback, and improves skills or agent instructions.

Typical Work Loop

The same loop works for valuation, capital, assumptions, experience studies, reinsurance, ALM, reporting, or a completely new data pack.

State intent

The actuary asks Copilot for an outcome and points to files, folders, or prior wiki pages.

Gather evidence

Copilot reads the wiki, profiles data, converts documents, and identifies gaps or contradictions.

Extend capability

If existing tools are not enough, Tool Builder drafts a focused Python tool and reviewer checks it.

Compound learning

Accepted outputs, decisions, corrections, and questions are filed back into wiki and feedback memory.

Example Team Conversation

Actuary prompt in VS Code

I uploaded a new capital data pack in raw/2026Q1-capital. Understand what is inside, update the wiki, propose useful Snowflake objects, and tell me what analysis tools are missing.

Expected agent behavior

  • Reads wiki schema, index, and recent log entries.
  • Profiles files and converts documents only where useful.
  • Creates source, metric, method, question, and decision pages.
  • Proposes Snowflake objects without assuming a fixed schema.
  • Builds new tools only when the analysis needs repeatable execution.
  • Records corrections as feedback and updates skills or tests.
The design principle: Copilot decides; MCP and Python assist; the wiki and feedback layer make the team smarter over time.
